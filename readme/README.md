# **Тестовое задание для Frontend-разработчика**


**Цель задания:** Создание адаптивной, интерактивной страницы с использованием современных технологий и сложных скролл-анимаций.

**Макет:** [Figma](https://www.figma.com/design/lKbrAMzPBLcqcjfn1ljibD/iteco-test-2?node-id=0-1&t=pHQipX3nf5usQtYU-1) 


## Технические требования

* **Адаптивная и резиновая верстка:** Верстка должна быть выполнена на rem. Размер шрифта корневого элемента (<html>) должен корректироваться через CSS Media Queries для следующих контрольных точек: **375px**, **1280px**, **1920px**, **2560px**.

* **Pixel Perfect:** Визуальное соответствие предоставленному макету Figma с погрешностью до 5px

* **Поддержка браузеров:** Chrome/Edge (Chromium-based), Firefox, Safari — последние две стабильные версии.

* **Стек технологий:**
  * **Обязательно:** JavaScript/TypeScript, SCSS.
  * **Приветствуется :** Vue.js 3 (Composition API) или Astro (с островной архитектурой).
  * **Сборка:** Vite.
  * **Анимации:** GSAP (для скролл-триггеров и сложных последовательностей) и нативная CSS-анимация.
  * **Видео:** Использовать корпоративную библиотеку/плеер (в рамках теста достаточно использовать тег <video> с корректными атрибутами контроля).
  * **Качество кода:** Обязательное использование линтеров (ESLint для JS/TS, Stylelint для SCSS/CSS) с конфигурацией, соответствующей выбранному фреймворку.

## Описание и поведение блоков

**Блок 1 (Hero-экран):**

* Занимает **100% высоты окна устройства (100vh)**.

* Содержит фоновое видео на цикле (loop, autoplay, muted).

* При скролле вниз активируется анимация:
  * Видео плавно сдвигается влево (например, с помощью transform: translateX).
  
  * Справа начинает "выезжать" текстовый контент (заголовок, описание, кнопка-указатель на скролл). Появление контента привязано к перемещению скролла.

* Шапка - **Логика меню и поиска не требуется, только верстка.**

* Навигационная панель внизу экрана в начале полная.

* При скролле вниз панель трансформируется в минималистичный вид (как в макете) с фиксированным позиционированием (position: fixed).

* Эта панель следует за пользователем до **конца первого блока**, после чего остается у его нижней границы при переходе ко второму блоку.

**Блок 2 (Parallax-зона):**

* Использовать эффект параллакса для фоновых элементов (изображений, слоев) при скролле. Скорость движения фона и переднего плана должна различаться.
* Контентная часть (текст, цифры) остается стабильной.

**Блок 3 (Фиксируемые карточки):**

* При достижении блока скроллом, он **фиксируется (position: sticky или аналог через JS)** на области просмотра.
* Внутри блока, при дальнейшем вертикальном скролле, происходит горизонтальная прокрутка карточек.
* При переключении между карточками используется fade-эффект (исчезновение одной и появление другой).

**Блок 4 (Появление элементов):**

* Элементы (например, иконки с текстом, описывающие "Преимущества системы") первоначально скрыты.
* По мере скролла до этого блока элементы плавно проявляются (opacity от 0 до 1) и одновременно "придвигаются" снизу (transform: translateY).


## **Процесс выполнения**

Перед началом работы необходимо:


1. **Проанализировать задачу:** Изучить это ТЗ, получить доступ к макету [Figma](https://www.figma.com/design/lKbrAMzPBLcqcjfn1ljibD/iteco-test-2?node-id=0-1&t=pHQipX3nf5usQtYU-1) (запросив доступ в dev режиме).
2. **Задать уточняющие вопросы:** По деталям анимаций, отсутствующим в макете состояниям, формату видео и т.д.
3. **Оценить объем и сроки:** Составить краткий план работы и согласовать срок сдач
4. Сдать результат через загрузку на любой бесплатный хостинг, после выгрузить в gitlab или передать архив.

## **Критерии оценки**

* **Соответствие макету (Pixel Perfect)** и корректная адаптивность.
* **Качество и семантичность верстки** (HTML5, доступность).
* **Корректность, производительность и плавность всех анимаций.**
* **Чистота и структурированность кода** (TypeScript, модульный SCSS, соблюдение стиля).
* **Настройка проекта** (Vite, линтеры, организованная файловая структура).
* **Соответствие техническим требованиям** (использование rem, коррекция кегля, поддержка браузеров).
